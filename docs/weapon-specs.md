### 4.3 선형 C2 킬체인 흐름 (KAMD 축 예시)

> **지연 모델**: S2S = Σ(링크지연) + Σ(처리지연). 링크지연은 데이터링크 통신 전송 시간, 처리지연은 C2 노드 내부 분석·판단 시간. 둘은 별개이며 합산.

```
[GREEN_PINE] ─16s링크→ [KAMD_OPS] ─16s링크→ [ICC] ─1s링크→ [ECS] ─1s링크→ [발사대]
                        (20~60s처리)          (5~15s처리)    (2~5s처리)

링크지연 합계: 16 + 16 + 1 + 1 = 34초 (고정)
처리지연 합계: (20~60) + (5~15) + (2~5) = 27~80초 (가변)
총 S2S: 61~114초

※ ICC로 정보가 전달되는 시점에서 포대의 MFR도 동시에 가동하여 탄도미사일 궤적 추적
※ 사령부(KAMD_OPS) 분석+장거리 링크 지연이 S2S의 60% 이상을 차지
```

### 4.4 Kill Web 킬체인 흐름

```
[모든 센서] ─1s링크→ [IAOC] ─1s링크→ [EOC] ─1s링크→ [사수]
                     (1~3s처리)       (1~3s처리)

링크지연 합계: 1 + 1 + 1 = 3초 (고정)
처리지연 합계: (1~3) + (1~3) = 2~6초 (가변)

※ IFCN 자동 전송, 계층적 승인 불필요
※ 선형 대비 S2S 85~92% 단축
```

P_detect = max(0, 1 - (d / R_eff)²) × (1 - jam) × det_factor

## 6. 교전 모델 — 2단계 구조

> v2.0에서는 요격미사일이 PNG 유도로 **물리적으로 비행**한다.
> 따라서 교전 모델을 **의사결정(교전할 것인가)**과 **결과 판정(요격 성공인가)**으로 분리한다.

### 6.1 예측 Pk (의사결정용)

교전 여부를 판단할 때 사용. **예측 요격 지점(predicted intercept point)**에서의 거리로 계산.
현재 사수↔위협 거리가 아니라, 요격미사일이 도달할 것으로 예측되는 지점에서의 조건.

```
predicted_Pk = base_pk × range_factor × maneuver_penalty × jamming_penalty

d_intercept = 예측 요격 지점에서의 사수↔위협 경사거리 (km)
range_factor = max(0, 1 - (d_intercept / R_max)²)
maneuver_penalty = 0.85 (위협이 종말단계 기동 중일 경우)
jamming_penalty = 1 - jamming_level

사용처: 교전 판정 (섹션 7.2)에서 threshold 비교
예: predicted_Pk ≥ 0.30 → 교전 승인
```

예시:
```
L-SAM ABM탄 vs SRBM:
- 위협 현재 고도 150km, 하강 중
- 예측 요격 지점: 고도 50km (교전구역 내), 경사거리 약 55km
- base_pk = 0.85
- range_factor = 1 - (55/150)² = 0.87
- maneuver_penalty = 0.85 (종말단계 기동)
- jamming_penalty = 1.0 (재밍 없음)
- predicted_Pk = 0.85 × 0.87 × 0.85 × 1.0 = 0.63 → 0.30 이상 → 교전 승인
```

### 6.2 요격 결과 판정 (물리 시뮬레이션)

실제 교전 결과는 **요격미사일의 물리적 비행**으로 결정.

```
1. 요격미사일 발사 → PNG 유도 비행 (physics.js pngGuidance)
2. 매 프레임: 요격미사일↔위협 거리 계산
3. 거리 ≤ kill_radius → 탄두 효과 판정 진입
4. 거리 > kill_radius 이고 요격미사일 연료 소진 또는 이탈 → MISS

탄두 효과 판정 (kill_radius 이내 도달 시):
  hit_probability = warhead_effectiveness × proximity_factor
  proximity_factor = 1.0 - (closest_distance / kill_radius)²
  Bernoulli(hit_probability) → HIT or MISS
```

요격방식별 파라미터:

| 요격방식 | kill_radius | warhead_effectiveness | 설명 |
|---------|-------------|---------------------|------|
| hit-to-kill | 50m | 0.95 | 직접 충돌 파괴 (THAAD, L-SAM ABM탄). 정밀도 요구 높음 |
| direct_hit | 200m | 0.90 | 위력증강형 탄두 직격 (천궁-II, PAC-3). 근접 폭발 효과 포함 |
| guided | 500m | 0.75 | 유도 + 근접신관 (천궁-I, L-SAM AAM탄, 비호, 천마). 파편 살상 |

**v0.7.3과의 차이**: v0.7.3은 요격미사일 엔티티 없이 Pk 확률만으로 즉시 판정.
v2.0은 요격미사일이 물리적으로 비행하여 도달 여부가 먼저 결정되고, 도달 시에만 탄두 효과 확률 적용.
→ 위협의 회피기동, 요격미사일 유도 성능, 교전 기하학이 결과에 실질적으로 영향.

## 7. 교전 정책 (ENGAGEMENT_POLICY)

### 7.1 발사 시점 결정 (선제 발사 원칙)

> **핵심 개념**: 요격미사일은 적 위협이 교전고도에 도달하기 **전에** 발사한다.
> 교전고도(예: L-SAM 60~100km)는 **요격이 이루어지는 고도**이지, 발사 결심 고도가 아니다.

```
발사 시점 결정 로직:
  1. 위협의 현재 위치·속도·궤적으로 미래 위치 예측
  2. 위협이 해당 사수의 교전구역(engagement_zone)에 도달하는 시간(t_threat) 계산
  3. 요격미사일이 예측 요격 지점까지 비행하는 시간(t_interceptor) 계산
  4. 발사 시점 = t_threat - t_interceptor - safety_margin
  5. 현재 시뮬레이션 시간이 발사 시점에 도달하면 → 교전 판정 진입 (7.2)

예시 (L-SAM ABM탄 vs SRBM):
  - SRBM 현재 고도 150km, 하강 속도 ~2km/s
  - L-SAM 교전구역 상한 100km 도달까지: (150-100)/2 = ~25초
  - L-SAM 요격미사일 비행시간: ~20~30초 (부스터+유도)
  - → 위협이 고도 ~130km일 때 발사 시점 도래 → 교전 판정 진입
```

### 7.2 교전 판정 흐름 (순차적 의사결정)

```
[STEP 1] 위협 우선순위 정렬
  SRBM > CRUISE_MISSILE > AIRCRAFT > MLRS_GUIDED > UAS
  탄도탄 우선 원칙: 다중임무 체계는 탄도탄 교전 최우선

[STEP 2] 해당 사수의 교전구역 판정
  위협이 이 사수의 교전구역(min/max range, min/max altitude)에
  진입할 예정인가? → NO → 이 사수는 후보에서 제외
  
[STEP 3] 발사 시점 판정 (7.1)
  현재 시각이 발사 시점에 도달했는가? → NO → 대기 (아직 이름)

[STEP 4] 예측 Pk 판정 (6.1)
  예측 요격 지점에서의 predicted_Pk 계산
  
  분기:
  a) predicted_Pk ≥ 0.30 → 교전 승인 → [STEP 5]로
  b) predicted_Pk < 0.30 이지만 잔여 교전 기회 ≤ 2 → 긴급 교전
     predicted_Pk ≥ 0.10이면 교전 승인 → [STEP 5]로
  c) predicted_Pk < 0.10 → 이 사수로는 교전 불가, 다른 사수 탐색

[STEP 5] 발사 → 물리 시뮬레이션 (6.2)
  요격미사일 생성 → PNG 유도 비행 → 탄두 효과 판정
```

**must_engage_distance 제거**: 기존 "30km 이내 무조건 교전"은 사수별 교전구역이
다르므로 보편 적용 불가 (L-SAM ABM탄의 최소 교전고도 60km에서 30km는 이미 구역 이탈).
대신 **STEP 2의 교전구역 판정**과 **STEP 4의 잔여 교전 기회** 로직이 최후 방어를 담당.

### 7.3 동시교전 제한

| 위협 유형 | 최대 동시교전 사수 수 |
|----------|-------------------|
| SRBM | 3 |
| CRUISE_MISSILE | 2 |
| AIRCRAFT | 2 |
| MLRS_GUIDED | 1 |
| UAS | 1 |

### 7.4 다중교전 시 결과 처리
- 동일 위협에 복수 사수가 교전 시, 각 요격미사일이 **독립적으로 물리 비행**
- 하나라도 탄두 효과 HIT → 위협 격추
- 다층 핸드오프: 교전 시도한 사수 유형 기록, 동일 유형 재교전 방지, 다른 유형은 허용
- 예: L-SAM ABM탄 MISS → 천궁-II direct_hit로 재교전 가능

### 7.5 다축 중복교전 (선형C2 전용)
- 동일 위협이 MCRC축·KAMD축 양쪽에서 탐지 시 각 축에서 독립 킬체인 실행
- 축 간 교전상태 미공유 (KAMD_OPS↔MCRC 간 링크지연 **16s**) → 중복교전 발생
- **Kill Web에서는 IAOC 통합 관리로 중복교전 없음** (핵심 가설2 검증 대상)

### 7.6 적응형 교전 정책 (Kill Web 전용)
탄약 소진에 따라 교전 기준을 자동 조정:
- **탄약 > 30%**: 표준 정책 (위협별 최대 동시교전 적용)
- **탄약 10~30%**: 단일 교전만 허용 (1표적 1발)
- **탄약 ≤ 10%**: 고위협(SRBM)만 1기 교전, 나머지 위협은 다른 사수에게 위임
- 선형 C2에서는 적응형 정책 미적용 (축간 정보 공유 부족으로 전체 탄약 상태 파악 불가)

## 8. 통신 채널 모델 (CommChannel)

### 8.1 선형 C2 데이터링크 지연

> **장거리 링크**: 탐지 전용 레이더↔사령부, 사령부↔대대 간 = **16초**
> **단거리 링크**: ICC↔ECS, ECS↔포대, 포대 내부(MFR↔ECS) = **1초 이내**

| 링크 | 경로 | 링크지연 | 구분 |
|------|------|---------|------|
| 조기경보→사령부 | GREEN_PINE → KAMD_OPS | **16s** | 장거리 |
| 조기경보→사령부 | EWR → MCRC | **16s** | 장거리 |
| 감시레이더→사령부 | FPS117 → MCRC | **16s** | 장거리 |
| 사령부→대대 | KAMD_OPS → ICC | **16s** | 장거리 |
| 사령부→대대 | MCRC → ICC | **16s** | 장거리 |
| 축간 정보공유 | KAMD_OPS ↔ MCRC | **16s** | 장거리 (중복교전 원인) |
| 대대→포대 | ICC → ECS | **1s** | 단거리 |
| 포대 내부 | MFR → ECS | **0.5s** | 직결 |
| 포대→발사대 | ECS → 발사대 | **1s** | 단거리 (사격케이블) |
| 국지방공 | 비호/천마 자체 | **0s** | C2 체인 외부, 독자 운용 |

### 8.2 Kill Web 데이터링크 지연 (IFCN)

> IFCN은 자가치유 메시 네트워크로, 장거리도 자동 라우팅하여 **1초 이내** 전송.

| 링크 | 경로 | 링크지연 | 설명 |
|------|------|---------|------|
| 센서→IAOC | 모든 센서 → IAOC | **1s** | IFCN 자동 전송, SHORAD/TPS880K 포함 |
| IAOC→EOC | IAOC → EOC | **1s** | 최적 사수 배정 |
| EOC→사수 | EOC → 발사대 | **1s** | 비호/천마 포함 |
| EOC↔EOC | 모든 EOC 간 | **0.5s** | 동일 항공상황도 동기화 |

### 8.3 재밍 열화 모델
```
link_degradation = base × (0.5 + random(0~1.0))
effective_delay = base_delay × link_degradation
if degradation > 0.8 → 링크 두절 (Infinity)
Kill Web: degradation × 0.5 (redundancy_factor, IFCN 다중경로)
```

### 8.4 EW 3단계

| 단계 | jamming_level | 탐지확률 감소 | 링크 영향 |
|------|-------------|------------|----------|
| LOW | 0.1 | -10% | 약간 지연 |
| MEDIUM | 0.3 | -30% | 일부 두절 (사령부→대대 링크 취약) |
| HIGH | 0.5 | -50% | 다수 두절, 선형C2에서 사령부↔대대 연결 거의 마비 |

## 11. 토폴로지 관계 정의 (TOPOLOGY_RELATIONS)

> weapon-data.js의 relations 필드로 구현. 새 체계 추가 시 여기에 관계만 선언하면
> registry.js가 자동으로 토폴로지를 생성.

### 11.1 선형 C2 축별 임무 분장

> **핵심 원칙**: 탄도미사일은 KAMD 담당, 그 외 위협(순항미사일/항공기/UAS)은 MCRC 담당.
> **교전 우선순위**: 탄도탄으로 식별된 위협은 최우선 방어. 다중임무 체계가 탄도탄과 비탄도탄을 동시에 대응해야 할 경우 탄도탄 교전이 먼저.

```
[KAMD축 — 탄도미사일 전담]
KAMD_OPS → ICC → ECS → THAAD
                       → L-SAM (ABM탄, 탄도미사일 임무)  ← ★ 중복 통제
                       → 천궁-II (탄도미사일 임무)        ← ★ 중복 통제
                       → PAC-3 (탄도미사일 임무)          ← ★ 중복 통제

[MCRC축 — 순항미사일/항공기/UAS 전담]
MCRC     → ICC → ECS → 천궁-I, KF-16
                       → L-SAM (AAM탄, 항공기/순항 임무)  ← ★ 중복 통제
                       → 천궁-II (항공기/순항 임무)        ← ★ 중복 통제
                       → PAC-3 (순항/항공기 임무)          ← ★ 중복 통제

[국지방공 — 선형 C2 미통합]
비호, 천마: 자체 레이더로 근접 위협 독자 탐지·교전 (상위 C2 명령 없이 자체 판단)
```

### 11.2 통제 중복 문제 (핵심 연구질문)

| 사수 | 교전가능 위협 | KAMD 통제 | MCRC 통제 | 통제 중복 | 비고 |
|------|-------------|----------|----------|---------|------|
| **L-SAM** | **SRBM(ABM탄), AIRCRAFT/CM/UAS(AAM탄)** | **✅ ABM탄** | **✅ AAM탄** | **⚠️ 중복** | 단일 포대(MFR+ECS) 양쪽 축 명령 수신 |
| **천궁-II** | **SRBM, AIRCRAFT, CM, UAS** | **✅ 탄도탄** | **✅ 항공기/순항/UAS** | **⚠️ 중복** | 단일 포대(MFR+ECS) 양쪽 축 명령 수신 |
| **PAC-3** | **SRBM, CM, AIRCRAFT** | **✅ 탄도탄** | **✅ 순항/항공기** | **⚠️ 중복** | 단일 포대(레이더+ECS) 양쪽 축 명령 수신 |
| THAAD | SRBM | ✅ | — | 없음 | 탄도탄 전용 |
| 천궁-I | AIRCRAFT, CM | — | ✅ | 없음 | 대항공기 전용 |
| KF-16 | AIRCRAFT, CM, UAS | — | ✅ | 없음 | 대항공기 전용 |
| 비호 | AIRCRAFT, UAS, CM | — | — | — | 선형: C2 미통합 (독자 국지방공) |
| 천마 | AIRCRAFT, UAS, CM | — | — | — | 선형: C2 미통합 (독자 국지방공) |

**선형 C2에서의 구조적 문제**:
- L-SAM/천궁-II/PAC-3 포대의 ECS가 KAMD축과 MCRC축 **양쪽에서 명령**을 받음
- 탄도미사일 요격 중에 항공기 교전 명령이 들어오면 → **우선순위 충돌** (탄도탄 우선 원칙으로 항공기 교전 지연)
- 축 간 교전상태 공유 지연(10~30s) → 동일 위협에 양축이 각각 교전 시도 → **중복교전**
- 탄도탄과 비탄도탄이 동시에 날아오면 한정된 MFR 추적용량을 **양축이 경쟁**
- 비호/천마는 아예 C2 체인 밖 → 네트워크 자산으로 활용 불가, 방어 효율 저하

**Kill Web에서의 해결**:
- IAOC가 모든 위협을 통합 관리 → 축 구분 없이 위협별 최적 사수 배정
- L-SAM이 ABM탄으로 탄도미사일 요격 중 → 항공기는 천궁-I/KF-16에 자동 배정 (L-SAM의 AAM탄은 온존)
- 교전상태가 전 네트워크에 실시간 공유 → 중복교전 원천 차단
- **비호/천마도 IAOC/EOC에 통합** → 근접 방어에서 UAS 포화 대응 시 네트워크 자산으로 조율 가능

### 11.3 센서 → C2 보고 관계

| 센서 | 보고 대상 (선형) | 역할 | 비고 |
|------|---------------|------|------|
| GREEN_PINE | **KAMD_OPS** (직접) | 조기경보 + 탄도탄 추적 | 사령부가 분석 후 ICC/ECS에 하달 |
| EWR | **MCRC** (직접) | 장거리 조기경보 | 사령부가 분석 후 ICC에 하달 |
| FPS117 | **MCRC** (직접) | 고고도 감시 | — |
| PATRIOT_RADAR | **ECS** (포대 소속) | PAC-3 포대 자체 화력통제 레이더 | — |
| MSAM_MFR | **ECS** (포대 소속) | 천궁-II/L-SAM 포대 자체 MFR (듀얼모드) | ICC에서 요격명령 수신 시 동시 가동 |
| SHORAD_RADAR | **자체** (포대 소속) | 비호/천마 자체 레이더 | 선형: 상위 C2 미연동, 독자 운용 |

### 11.4 Kill Web 모드에서의 관계
- **모든 센서** → IAOC (직접 보고, IFCN 자동 전송) — SHORAD_RADAR, TPS880K **포함**
- IAOC → EOC (최적 사수 배정, 축 구분 없이 위협 유형별 최적화)
- EOC → 사수 (직접 통제) — **비호/천마 포함**
- 모든 EOC가 동일 항공상황도 유지 — 어느 EOC에서든 어느 사수든 통제 가능
- **통제 중복 해소**: L-SAM/천궁-II/PAC-3이 IAOC 통합 관리 하에 탄종별 최적 배정
- **국지방공 통합**: 비호/천마가 네트워크 자산으로 편입 → UAS 포화 등에 조율된 대응
- **EOC = ICC + ECS 기능 통합** (IBCS가 기존 계층적 C2를 대체)

```
LAMD: {

                pkTable: { MLRS_GUIDED: 0.80, CRUISE_MISSILE: 0.60 },
                ammoCount: 16, interceptMethod: 'direct_hit' },
  relations: {
    ecs: 'ECS',
    icc: 'ICC',
    commandC2: ['KAMD_OPS'],              // 탄도 시그니처 위협 담당
    c2Axis: 'KAMD',
    engageableThreats: ['MLRS_GUIDED', 'CRUISE_MISSILE'],
    requiredSensors: ['MSAM_MFR', 'TPS880K']
  }
}
```
→ weapon-data.js에 이 항목만 추가하면 registry.js가 자동으로
  KAMD_OPS→ICC→ECS 체인에 연결, 킬체인에서 MLRS_GUIDED 교전 시 사수 후보로 등장.
