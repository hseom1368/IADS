# IBCS 개념 요약 — 시뮬레이션 설계 근거

> 이 문서는 IBCS(Integrated Battle Command System)의 핵심 개념을 정리하여
> Kill Web 아키텍처 시뮬레이션의 설계 근거로 활용합니다.

---

## 1. IBCS란?

Northrop Grumman이 개발한 미 육군 통합방공미사일방어 지휘통제체계.
8개 레거시 C2 시스템을 단일 소프트웨어 정의 아키텍처로 통합.
핵심 운용 개념: **"Any Sensor, Best Shooter"**

- 2023년 4월 완전율 생산 승인, 5월 IOC 달성
- 비행시험 성적: **32전 32승** (실사격 테스트)
- 폴란드 2025년 12월 FOC 선언 (최초 해외 운용국)
- 총 투자: 40억 달러 이상

## 2. 킬체인 vs 킬웹 — 핵심 차이

### 선형 킬체인 (Legacy)
```
탐지 → 추적 → 식별 → 결심 → 교전
(단일 센서)  (단일 C2)  (단일 사수)
```
- 각 단계가 순차 실행, 한 링크 실패 시 전체 실패
- 계층적 C2 체인: 사령부(KAMD_OPS/MCRC) → ICC(대대) → ECS(포대)
- 장거리 링크지연 16s × 2구간 + 처리지연 합산 → S2S 시간 **61~114초** (KAMD축)
- MCRC/KAMD/육군방공 3축 분리 → 통제 중복 문제

### IBCS 킬웹
```
    [센서A]─┐
    [센서B]─┤──→ [IAOC 융합] ──→ [EOC 최적사수] ──→ [교전]
    [센서C]─┘     (1~3초처리)      (1~3초처리)
```
- 다중 경로 동시 존재, 한 노드 실패 시 자동 우회
- IFCN 링크지연 1s × 3구간 + 처리지연 → S2S 시간 **5~9초**
- EOC = ICC+ECS 기능 통합. 축 구분 없음

## 3. 시뮬레이션에서 모델링해야 할 IBCS 핵심 기능

### 3.1 컴포지트 트래킹 (Composite Tracking)
- 다중 센서의 raw 데이터를 융합하여 단일 통합 항적(SIAP) 생성
- 오차 감소: `fused_error = base_error / √N` (N = 추적 센서 수)
- 하한값: 0.1km (최소 오차)
- 개별 센서 강점 활용 (EWR: 원거리, MFR: 정밀 추적, FPS117: 고고도)
- 시뮬레이션 효과: Pk 보너스 최대 +10%

### 3.2 위협 식별 정확도
- **선형 C2**: 단일 센서 기반 → 탄도 시그니처의 MLRS를 70% 확률로 SRBM 오인식
- **Kill Web**: 2개+ 센서 추적 시 100% 정확 식별, 단일 센서 시에도 10%만 오인식
- 효과: 고가 요격탄(PAC-3) 낭비 방지

### 3.3 자동 최적 사수 선정
Kill Web의 사수 점수 공식:
```
score = base_pk × (1/거리) × 탄약비율 × 부하계수 + 아군상태보너스(0.15)
```
- **base_pk**: 무기체계 고유 Pk (pkTable에서 위협 유형별 조회). predicted_Pk가 아님 — 거리가 별도 항으로 존재하므로 이중 계산 방지
- 거리, 잔여 탄약, 현재 교전 부하를 종합 고려
- COP(Common Operational Picture) 기반 실시간 아군 상태 공유

### 3.4 적응형 교전 정책
- 탄약 30% 이하: 단일 교전만 허용
- 탄약 10% 이하: 고위협(SRBM)만 1기 교전
- 정상: 표준 정책 (SRBM 최대 3기 동시교전)

### 3.5 IFCN 자가치유 메시 네트워크
- 전술 무선, 광섬유, 위성통신, 4G/5G 다중 베어러
- 노드 파괴/재밍 시 자동 경로 재설정
- 모든 EOC가 동일한 항공상황도 동시 유지
- 단일 장애점(SPOF) 없음

## 6. 시뮬레이션 설계 시사점

1. **네트워크가 무기체계**: 개별 레이더/미사일보다 조합·분해·재조합 능력이 핵심
2. **방어 자산 40% 증대**: 동일 병력으로 IBCS 적용 시 방어 범위 40% 확장 (NG 공식)
3. **단발 교전 교리**: 컴포지트 트래킹 정확도로 "1표적 1발" 가능 → 탄약 효율 2배
4. **회복탄력성**: 노드 파괴 시 대체 경로 자동 활성화 → 누출률 감소
5. **S2S 시간 단축**: 5~9초 vs 61~114초 → 빠른 위협 대응
