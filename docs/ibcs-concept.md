## 2. 킬체인 vs 킬웹 — EADSIM-Lite 모델링 매핑

### 선형 킬체인 (Legacy) → LinearKillChain

```
GREEN_PINE 탐지 → 탐지→추적(10s)
  ↓ (16s 링크)
KAMD_OPS: 시스템(5~10s) + 운용원(15~50s)
  ↓ (16s 링크)
ICC: 시스템(3~5s) + 운용원(2~10s)
  ↓ (1s 링크)
ECS: 시스템(1~2s) + 운용원(1~3s) → MFR 교전급 확립(8s)
  ↓ (1s 링크)
발사 → PSSEK 5단계 → S-L-S/BDA

총 S2S: 84~137초 (고숙련~저숙련)
```
- 각 단계 순차, 한 링크 실패 → 전체 실패
- KAMD/MCRC/국지방공 3축 분리 → 통제 중복
- 단일 센서 → MLRS 70% SRBM 오인식 → 고가탄 낭비
- 오상관 5%, 미상관 10% → 중복교전

### IBCS 킬웹 → KillWebKillChain

```
모든 센서 → IAOC 컴포지트 트래킹(1s)
  → 자동 최적 사수 선정(1~3s)
  → EOC(1~3s) → 발사

총 S2S: 5~9초
```
- 다중 경로 동시 존재, 노드 실패 시 자동 우회
- 컴포지트 트래킹: error = base/√N, **Pk +10%**
- 2개+ 센서 100% 정확 식별
- 오상관 1%, 미상관 2%
- 축 구분 없이 최적 사수 배정 → 통제 중복 해소

## 3. 시뮬레이션에서 모델링하는 IBCS 핵심 기능

### 3.1 컴포지트 트래킹 (Composite Tracking)
- 다중 센서 융합 → 단일 통합 항적(SIAP)
- `fused_error = base_error / √N`
- Pk 보너스: +10% (PSSEK에 곱셈 보정)
- EADSIM 모델링: sensor-model.js에서 IAOC가 자동 수행

### 3.2 위협 식별 정확도
- 선형: 단일 센서 → MLRS 70% SRBM 오인식
- Kill Web: 2개+ 센서 100% / 단일 10% 오인식
- 효과: 고가 요격탄(PAC-3 hit-to-kill) 낭비 방지

### 3.3 자동 최적 사수 선정
```
score = base_pk × (1/거리) × 탄약비율 × 부하계수 + friendly_bonus(0.15)
```
- **base_pk**: PSSEK 테이블 최대값 (최적 거리구간·정면 기준)
- 거리, 잔여 탄약, 현재 교전 부하, 동시교전 상한 종합 고려
- COP 기반 실시간 아군 상태 공유

### 3.4 적응형 교전 정책
- 탄약 > 30%: 표준 (위협별 최대 동시교전)
- 탄약 10~30%: 단일 교전만 (1표적 1발)
- 탄약 ≤ 10%: 고위협(SRBM)만 1기 교전
- 선형 C2: 적응형 미적용 (축간 탄약 상태 미공유)

### 3.5 IFCN 자가치유 메시 네트워크
- 전술 무선, 광섬유, 위성, 4G/5G 다중 베어러
- 노드 파괴/재밍 시 자동 경로 재설정
- 모든 EOC 동일 항공상황도
- EADSIM 모델링: comms.js에서 Kill Web `redundancy_factor = 0.5`

## 6. 시뮬레이션 설계 시사점

1. **네트워크가 무기체계**: 개별 센서/미사일보다 조합·분해·재조합 능력이 핵심
2. **방어 자산 40% 증대**: 동일 병력으로 IBCS 적용 시 방어 범위 40% 확장
3. **단발 교전 교리**: 컴포지트 트래킹으로 S-L-S 신뢰성 향상 → 탄약 절약
4. **회복탄력성**: IFCN 다중경로 → 누출률 감소
5. **S2S 단축**: 5~9초 vs 84~137초
